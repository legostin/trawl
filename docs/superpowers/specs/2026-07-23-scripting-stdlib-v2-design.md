# Scripting stdlib v2: JSONPath-ядро, хелперы, подсказки, dry-run

Дата: 2026-07-23. Статус: дизайн утверждён, ждёт имплементационного плана.

## Мотивация

Правила сегодня требуют ручного цикла parse → мутация → serialize с защитными
проверками на каждом шаге. Реальный инцидент: правило
`GET /v3/adverts/recommendation` обращалось к несуществующему `response.data`
(поле есть только у `sendJsonRequest`), падало с `cannot read property of
undefined`, и клиент оставался без ответа. Задача типа «во всех элементах
массива проставить поле» должна решаться одной строкой, ошибки — быть
самодиагностирующими, а справка — жить в самом проекте.

## Решение в одном абзаце

Ядро — path-DSL по полному JSONPath (RFC 9535) на базе Rust-крейта
`serde_json_path`; JS-обвязка в stdlib применяет мутации по нормализованным
локациям, которые возвращает Rust. Плюс четыре домена хелперов (URL/query,
моки, генерация данных, коллекции), подсказки JSONPath в Monaco по структуре
реального трафика, валидация при сохранении, dry-run на захваченных flow,
единый манифест документации (UI + MCP + cookbook) и исчерпывающие тесты.
Поведение при ошибках — fail-closed (как сейчас), но с диагностическими
сообщениями и трассой выполнения.

## 1. Архитектура

- Новая зависимость: `serde_json_path` (RFC 9535, проходит официальный
  JSONPath compliance test suite).
- `STD_LIB` переезжает из минифицированной строки в `scripting.rs` в файл
  `src-tauri/js/stdlib.js` (читаемый JS с комментариями), подключается через
  `include_str!`. Поведение существующих функций не меняется.
- Новая нативная функция `__native_jsonpath_locate(doc_json, path)`
  регистрируется в обоих движках (request/response-фаза и handler).

Поток `patch(msg, 'items[*].price', fn)`:

1. `doc = jsonBody(msg)` — парс тела, кэш распарсенного дока на объекте
   сообщения (повторные `patch` в одном правиле не парсят заново);
2. `JSON.stringify(doc)` + путь → Rust `query_located()`;
3. Rust возвращает нормализованные локации:
   `[["items",0,"price"], ["items",1,"price"], …]`;
4. JS идёт по локациям, применяет значение или функцию-модификатор
   (замыкания остаются в JS, мост передаёт только данные);
5. `setJsonBody(msg, doc)` — сериализация обратно.

Эргономика: ведущий `$.` в путях можно опускать — `'items[*].price'`
нормализуется в `'$.items[*].price'`. Полный синтаксис RFC 9535 (`$..`,
срезы, фильтры `[?@.a && @.b]`, `length()`, `match()`) доступен как есть.

Один Rust-парсер обслуживает рантайм, валидацию при сохранении, dry-run и
подсказки в редакторе.

## 2. Поверхность API

Все функции принимают и сообщение (`request` / `response` / результат
`send`), и уже распарсенный объект — по типу аргумента.

### JSONPath-операции над телом

| Функция | Поведение |
|---|---|
| `patch(msg, path, valueOrFn)` | Записать значение / применить функцию во всех совпавших узлах. Возвращает число узлов. 0 узлов → исключение (fail-closed). |
| `tryPatch(msg, path, valueOrFn)` | То же, но 0 узлов допустимо. |
| `pick(msg, path)` | Массив всех совпавших значений. |
| `pickOne(msg, path)` | Первое значение или `null`. |
| `removeAt(msg, path)` | Удалить совпавшие узлы. Возвращает число удалённых. |
| `mergeAt(msg, path, obj)` | Deep-merge объекта в каждый совпавший узел. |

Пример (замена исходного падавшего правила):

```js
const res = send(request);
patch(res, 'items[*].advertData.addDateFormatted', nowISO());
return res;
```

### URL/query

`setQueryParam(req, name, value)`, `removeQueryParam(req, name)`,
`rewriteHost(req, host)`, `rewritePath(req, from, to)` (строка или RegExp),
`pathSegments(req)`. Существующий `queryParam` остаётся.

### Моки и ответы-сахар

`json(obj)` / `json(status, obj)`, `textResponse(status, body)`,
`httpError(status, msg?)` — обёртки над `ctx.mock`. `delay(ms)` — только в
handler-фазе (request-фаза исполняется на общем потоке движка, блокировать
его нельзя; ограничение документируется).

### Генерация данных

`uuid()`, `randomInt(a, b)`, `randomFrom(arr)`, `nowISO(shift?, tz?)` —
`nowISO('+2d', '+05:00')`; сдвиги `'-1h'`, `'+30m'`, `'+2d'`.

### Коллекции

`groupBy(arr, keyOrFn)`, `sortBy(arr, keyOrFn)`, `uniqBy(arr, keyOrFn)`,
`chunk(arr, n)`, `sample(arr, n)`.

Существующие функции stdlib не меняются — обратная совместимость полная.
Итого ~20 новых функций.

## 3. Подсказки JSONPath в Monaco

Переиспользуем существующий анализ структуры трафика (`analyzeJson` /
`FieldInfo` из `src/lib/analyze.ts`, который уже питает тип
`TrawlResponseData`). Три уровня, все — внутри строкового литерала-пути в
аргументах path-функций:

1. **Автокомплит сегментов**: `CompletionItemProvider` предлагает следующий
   сегмент из реальной структуры захваченных ответов, совпавших с паттерном
   правила; для массивов — сниппеты `[*]`, `[0]`, `[?...]`.
2. **Живая валидация синтаксиса**: debounced Tauri-команда
   `validate_jsonpath(path)` (тот же Rust-парсер) → маркеры ошибок под
   строкой до сохранения.
3. **Счётчик совпадений**: путь прогоняется через locate по телу последнего
   совпавшего flow; inlay-подсказка `→ 20 узлов` или предупреждение
   `→ 0 узлов — путь не совпадает с реальным трафиком`.

Определение «курсор в аргументе-пути» — разбором текущей строки (регэксп по
вызовам известных path-функций), без полного AST.

## 4. Ошибки, трасса, валидация, dry-run

**Fail-closed**: исключение в скрипте или `patch` с 0 узлов роняет flow в
`error` (текущее поведение сохраняется). Сообщение становится
диагностическим — имя правила, строка (из `lineNumber` QuickJS), операция и
срез структуры тела верхнего уровня:

```
rule "GET /v3/adverts/recommendation", строка 2:
patch('items[*].advertData.addDateFormatted'): 0 узлов.
Тело ответа: { status, limit, offset, nbTotal, items[20], … }
```

**Трасса выполнения**: правило пишет в flow список операций
`[{op:'patch', path:'…', nodes:20}, {op:'send', status:200, ms:294}]`.
Отображается в деталях flow в UI и отдаётся в MCP `get_flow`.

**Валидация при сохранении** (`save_rule`, UI и MCP): компиляция скрипта в
QuickJS (синтаксис JS) + прогон строковых литералов в аргументах
path-функций через Rust-парсер JSONPath. Невалидное правило не сохраняется,
ошибка возвращается с позицией.

**Dry-run**: кнопка «Проверить на трафике» в редакторе правила и MCP-инструмент
`test_rule(script | ruleId, flowId?)`. Берётся последний захваченный flow,
совпавший с паттерном (или указанный); правило исполняется в песочнице,
`send()` в handler-фазе реплеит захваченный ответ вместо сети. Результат —
diff тела до/после, трасса, ошибки. В сеть ничего не уходит, ничего не
сохраняется.

## 5. Справка

Единый манифест `src/scripting/stdlib-docs.ts` (категория, сигнатура,
описание RU, пример «задача → одна строка») питает трёх потребителей:

1. **UI «Function library»**: категории, примеры с кнопкой «вставить в
   правило», экран-шпаргалка по синтаксису JSONPath на один экран.
2. **MCP `get_scripting_reference`**: обновлённые `.d.ts` + манифест +
   раздел «частые ошибки» («`send()` не имеет `.data`», «мутация `res.data`
   без `setJsonBody` — no-op»).
3. **Cookbook `docs/scripting-cookbook.md`**: ~15 рецептов (мок эндпоинта,
   патч всех элементов, редирект на стейдж, эмуляция 500/медленной сети,
   извлечение токена в env, A/B-подмена и т.п.).

Monaco-декларации (`src/scripting/stdlib.ts` / `apiTypes.ts`) обновляются
вместе с библиотекой.

## 6. Тесты

- **Rust через реальный QuickJS**: каждая новая функция — позитив/негатив по
  существующему паттерну тестов в `scripting.rs`. JSONPath-интеграция:
  locate → мутация → сериализация на реальных телах (kolesa-ответ как
  фикстура).
- **Edge-кейсы**: пустое/не-JSON тело, unicode-ключи, `null` в середине
  пути, массив в корне, тело 1 МБ (перф-смоук), фильтры по отсутствующим
  полям, модификатор, бросающий исключение.
- **RFC-соответствие**: compliance suite гоняет сам `serde_json_path`; мы
  покрываем свою обвязку (нормализация `$.`, locate-мост).
- **Vitest**: синхронность `stdlib.js` ↔ `.d.ts` ↔ манифест доков (каждая
  функция объявлена везде); completion-провайдер и детект «курсор в пути»;
  dry-run UI-логика.
- **Tauri-команды**: `validate_jsonpath`, `test_rule` (включая реплей
  `send`).

## Вне скоупа

- Декларативные правила без JS (JSON-спека операций) — отдельный проект,
  если понадобится.
- `delay()` в request-фазе (требует ухода от общего потока движка).
- Изменение семантики существующих функций stdlib.

## Открытые решения, зафиксированные в ходе брейншторма

- Механика: path-DSL, а не fluent-цепочки и не декларативные правила.
- Синтаксис: полный JSONPath RFC 9535 (крейт, не самописный парсер).
- Ошибки: fail-closed; `patch` без совпадений — ошибка, мягкий вариант —
  `tryPatch`.
- Имена: `removeAt` / `mergeAt` (не `remove` / `merge`).
- Реализация: Rust locate + JS-мутации (подход 1), stdlib — в
  `src-tauri/js/stdlib.js` через `include_str!`.
