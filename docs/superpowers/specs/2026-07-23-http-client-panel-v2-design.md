# HTTP Client v0.6.0 — панель: вкладки, поиск, ресайз, персистентная история

**Дата:** 2026-07-23
**Репозиторий:** `trawl-plugin-http-client` (хост не затрагивается).

## Цели

1. Левая панель ресайзится по ширине перетаскиванием.
2. Панель разбита на вкладки **Collection | History | Env**.
3. Поиск по коллекции (фильтрованное дерево) и по истории.
4. История отправок переживает перезапуск: пер-проект, до 5000 записей, тела ответов не сохраняются.

## 1. Ресайз панели

- Drag-handle ~5px по правому краю панели (`cursor: col-resize`), pointer events + `setPointerCapture`.
- Диапазон 180–520px, дефолт 250. Ширина сохраняется в `H.storage` (`httpclient.panelWidth`, глобально, не пер-проект) при отпускании и восстанавливается при монтировании.
- Без сторонних библиотек (бандл плагина держим маленьким).

## 2. Вкладки панели

- Переключатель Collection | History | Env сверху панели (реюз внутреннего `Tabs`-хелпера).
- Активная вкладка персистится в `httpclient.panelTab`.
- **Collection**: поле поиска, кнопка «+ Folder», инлайн-флоу сохранения (имя + селект папки), дерево `CollectionTree`.
- **History**: поле поиска, кнопка «Clear» с двухкликовым подтверждением («Clear» → «Sure?»), список записей.
- **Env**: текущая секция Environment (RowList + Save) без функциональных изменений.

## 3. Поиск

- Отдельное значение поиска у Collection и History (state компонента, не персистится).
- **Коллекция**: чистая функция `filterCollection(c, query)` в `collections.ts` → `{ folders: Set<string>, items: Set<string> } | null` (null при пустом запросе = показывать всё). Правила (регистронезависимая подстрока):
  - запрос совпал по имени, URL или методу → виден запрос и все папки-предки;
  - папка совпала по имени → видна папка, все её предки и всё её поддерево (вложенные папки и запросы).
- При непустом запросе все видимые папки принудительно раскрыты (state `closed` игнорируется); меню ⋯ работает как обычно.
- **История**: подстрочный фильтр по URL, методу и `String(status)`.

## 4. Персистентная история

Новый модуль `history.ts` (по образцу `collections.ts`):

```ts
export interface HistoryEntry {
  ts: number;              // Date.now() отправки
  method: string;
  url: string;             // финальный URL (после env-подстановки)
  status: number;          // 0 при сетевой ошибке
  req: ClientRequest;      // снапшот редактора; multipartFiles без байтов
}
```

- Ключ `httpclient.history.<projectId|_global>`.
- `loadHistory(): Promise<HistoryEntry[]>`, `pushHistory(e): Promise<HistoryEntry[]>` (prepend + cap **5000** + notify), `clearHistory(): Promise<void>` (+ notify пустым), `onHistoryChange(cb): () => void`.
- Тело ответа не сохраняется нигде (в записи нет полей ответа, кроме `status`).
- Тела запросов не режутся — повторное открытие из истории точное.
- Пишут историю: `send` в UI и MCP-тул `send_request` (`url` в записи — из wire-запроса).
- UI: перезагрузка списка при смене проекта и по `onHistoryChange` (live-синк с MCP-отправками); рендер ограничен первыми **200** совпадениями с подписью «showing 200 of N»; тултип строки — дата/время из `ts`.
- Клик по записи — как раньше, открывает снапшот в новой вкладке.

## 5. Тесты и версия

- `history.test.ts`: пер-проектный ключ, prepend-порядок, cap 5000, notify + отписка, отсутствие полей ответа, clear.
- `collections.test.ts` (+ кейсы `filterCollection`): матч по имени/URL/методу с видимыми предками, матч папки открывает поддерево, пустой запрос → null, пустой результат.
- Версия плагина → **0.6.0** (`package.json`, `trawl-plugin.json`), `apiVersion` не меняется. README: раздел про панель/поиск/историю. Релиз: мерж в main, пуш, тег `v0.6.0`.

## Вне скоупа

- Виртуализация списка истории (ограничиваемся рендером 200 строк).
- Усечение тел запросов в истории (обсуждено — не режем).
- Экспорт/импорт истории, история в MCP-тулах.
