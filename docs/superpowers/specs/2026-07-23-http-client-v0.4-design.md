# HTTP Client v0.4.0 — MCP-тулы, папки, cURL, тумблер прокси

**Дата:** 2026-07-23
**Репозитории:** `trawl-plugin-http-client` (основная работа), `http-catch` (автостарт прокси).

## Цели

1. MCP-тулы плагина: полный спектр работы с клиентом (создание/сохранение/отправка запросов, анализ ответов) — headless + live-синк открытого UI.
2. Переименование сохранённых запросов.
3. Папки в коллекциях (вложенные, любой глубины).
4. Импорт cURL при вставке в адресную строку и экспорт «Copy as cURL».
5. «Via proxy» — тумблер вместо чекбокса; при включённом тумблере запрос идёт через прокси всегда: если прокси остановлен, хост автоматически стартует его перед отправкой.

## 1. Коллекции: формат v2

Ключ хранения прежний: `httpclient.collection.<projectId|_global>` (в `H.storage`).

```ts
interface CollectionV2 {
  version: 2;
  folders: { id: string; name: string; parentId: string | null }[]; // parentId — вложенность любой глубины
  items:   { id: string; name: string; folderId: string | null; request: ClientRequest }[];
}
```

- **Миграция при чтении:** если распарсенное значение — массив (v1), оно оборачивается в `{version: 2, folders: [], items: [...{folderId: null}]}`. Запись всегда в v2. Обратной миграции нет.
- **Операции** в `collections.ts`: `loadCollection`, `saveCollection`, `addToCollection(name, request, folderId?)`, `updateRequest(id, request)`, `createFolder(name, parentId?)`, `renameItem(id, name)` (запрос или папка), `moveItem(id, folderId|parentId)`, `deleteItem(id)`.
- **Удаление папки — рекурсивное** (вместе с подпапками и запросами); UI спрашивает подтверждение.
- **Защита от циклов:** `moveItem` отклоняет перемещение папки в саму себя или своего потомка.
- **Событие изменения:** после каждой записи `collections.ts` уведомляет локальных подписчиков (module-level `Set` колбэков). UI подписан — правки из MCP-тулов сразу видны в открытой вкладке.

## 2. UI коллекции

- Дерево: папки с раскрытием/сворачиванием (состояние раскрытия — в памяти компонента), отступы по глубине, запросы внутри.
- Кнопка **«+ Folder»** рядом с «+ Save» — инлайн-поле имени (как текущий save-flow). Папка создаётся в корне; вложенные — через «Move to…».
- У каждой строки (запрос и папка) меню **«⋯»**: Rename (инлайн-поле), Move to… (выпадающий список всех папок + корень; для папок — без себя и потомков), Delete (для папок — confirm).
- Save-flow дополняется выбором папки: рядом с полем имени — селект папки (по умолчанию корень).

## 3. Тумблер прокси и автостарт (хост + плагин)

- **Хост, `src/plugins/host.ts`:** обёртка `http.send(req, viaProxy)` при `viaProxy === true` проверяет прокси-стор; если `running === false` — вызывает `startProxy(8729)` и обновляет состояние стора (UI хоста показывает прокси запущенным). Ошибка старта (порт занят и т.п.) пробрасывается как ошибка отправки.
- **Плагин:** чекбокс «via proxy» заменяется визуальным тумблером (мини-компонент на span/див, стили в духе текущих inline-стилей). Состояние персистится в `H.storage` (`httpclient.viaProxy`, глобально, не per-project) и восстанавливается при монтировании.

## 4. cURL импорт/экспорт

Новый модуль `curl.ts` в плагине.

### `parseCurl(text): ClientRequest | null`

- Возвращает `null`, если текст не начинается с `curl` (после trim) или не парсится.
- Шелл-токенизация: одинарные/двойные кавычки, экранирование, `\` + перенос строки, `$'...'` — как обычная строка.
- Опции: `-X/--request`, `-H/--header` (заголовки; `Cookie` остаётся заголовком), `-d/--data/--data-raw/--data-binary/--data-ascii` (несколько объединяются через `&`; если Content-Type не задан или `application/x-www-form-urlencoded` — раскладывается в form-режим, иначе raw), `--data-urlencode` (form), `-F/--form` (multipart; значения `@file` создают файловую часть с именем, но без байтов — файл перевыбирается вручную), `-u/--user` (заголовок `Authorization: Basic base64(...)`), `-b/--cookie` (заголовок `Cookie`), `--url`, позиционный URL. `--compressed`, `-k/--insecure`, `-s`, `-L` — игнорируются молча; неизвестные опции игнорируются.
- Метод по умолчанию: `POST`, если есть тело, иначе `GET`. Query из URL раскладывается в Params (существующий `parseUrl`).

### `toCurl(req, env): string`

- Многострочный вывод с `\`, значения в одинарных кавычках с экранированием.
- `{{vars}}` подставляются из env (исполняемая команда).
- form → `--data-urlencode`, multipart → `-F` (`-F 'name=@fileName'` для файлов), raw → `--data-raw`. Заголовки — только enabled-строки; disabled params не попадают в URL.

### Интеграция

- **Импорт:** обработчик paste на URL-инпуте; если вставляемый текст после trim начинается с `curl ` — `preventDefault`, парсинг, полное замещение текущего запроса, тост «Imported from cURL». Если парсинг вернул `null` — обычная вставка.
- **Экспорт:** кнопка «Copy as cURL» рядом с Send; копирует в буфер, тост-подтверждение.

## 5. MCP-тулы

Регистрация в `plugin.tsx` при инициализации бандла (единственное окно, когда мост разрешает `mcp.registerTool`); обработчики — в новом `mcpTools.ts`. Имена в MCP: `http-client_<name>`.

| Тул | Вход | Выход |
| --- | --- | --- |
| `send_request` | либо `savedId`, либо поля запроса (`method`, `url`, `params?`, `headers?`, `bodyMode?`, `rawBody?`, `form?`); `viaProxy?` (default false), `substituteEnv?` (default true) | `status`, `headers`, `body` (>200 КБ — обрезка + `truncated: true`), `durationMs`, `error?` |
| `list_collection` | — | дерево: папки (id, name, parentId) + запросы (id, name, folderId, method, url) |
| `get_request` | `id` | сохранённый запрос целиком |
| `save_request` | `name`, поля запроса, `folderId?`, `id?` (обновление существующего) | `id` |
| `update_item` | `id`, `name?`, `folderId?`/`parentId?` | ok |
| `delete_item` | `id` | ok (папки — рекурсивно, **без** подтверждения: MCP-клиент отвечает сам) |
| `create_folder` | `name`, `parentId?` | `id` |
| `set_draft` | поля запроса | ok; редактор UI обновляется live (или запрос ждёт открытия вкладки) |
| `get_draft` | — | текущий черновик редактора (или pending), `null`, если нет |
| `get_last_response` | — | последний ответ, отправленный из UI или через `send_request` (тот же формат, что у `send_request`), `null`, если нет |

- Headless: коллекции — через `collections.ts` (хранилище), отправка — `H.http.send` с подстановкой env активного проекта (`H.projects.active()?.env`); автостарт прокси наследуется от хоста.
- `state.ts` расширяется: module-level хранилище draft + последнего ответа; UI-компонент подписывается и публикует свои изменения туда же (двусторонний синк draft/response между UI и тулами).
- Ошибки (`id не найден`, невалидный URL и т.п.) — `throw new Error(<текст>)`; мост вернёт их MCP-клиенту.

## 6. Тесты

- Плагин получает vitest (dev-dependency + `test`-скрипт):
  - `curl.test.ts` — токенизация (кавычки, multiline `\`), все поддерживаемые опции, round-trip parse→toCurl, не-curl текст → `null`.
  - `collections.test.ts` — миграция v1→v2, папочные операции, рекурсивное удаление, защита от циклов при move (storage мокается).
- Хост: изменение `host.ts` минимально; ручная проверка автостарта (тумблер on + прокси остановлен → запрос уходит, прокси в топбаре становится running).

## 7. Версии и документация

- Плагин: `version` → `0.4.0` в `trawl-plugin.json` и `package.json`; README — раздел про MCP-тулы, cURL и папки.
- `apiVersion` не меняется (мост `mcp.*` уже входит в текущий host API).
- Хост: версия не меняется (правка `host.ts` обратно совместима).

## Вне скоупа

- Drag-and-drop в дереве коллекции (перемещение — через «Move to…»).
- Экспорт cURL с сохранением `{{vars}}` (подставляем значения; при необходимости — отдельной опцией позже).
- Байты файлов multipart в сохранённых запросах и cURL-импорте (как и раньше, перевыбор вручную).
